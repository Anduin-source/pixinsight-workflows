  ── 5.1 Decisão MGC vs GC (REVISADA para hemisfério sul) ─────────────────────

    Critério              Use MGC                           Use GC
    ──────────────────────────────────────────────────────────────────────────
    Telescópio            Epsilon 160 (wide-field)          RC 12" (objetos compactos)
    Alvo                  Nebulosas extensas, cirros,       Galáxias isoladas, aglomerados,
                          IFN, campos difusos               planetárias
    Fundo no quadro       Pouco fundo limpo disponível      Fundo dominante e limpo
    Declinação            ≥ -25°: tentar MGC, verificar     < -30°: GC é default.
                          cobertura MARS no tool.           -25° a -30°: verificar
                                                            cobertura caso a caso.
    Posição galáctica     Fora do plano (|b| > 10°)         Plano galáctico (|b| < 10°)
    Pré-requisitos        ImageSolver + SPFC                Nenhum adicional
                          + XMARS (~1.35 GB)
    Risco                 Pode falhar sem cobertura MARS    Baixo — fallback confiável

    PROCEDIMENTO PRÁTICO:
      1. Abrir MGC com a imagem aberta.
      2. Verificar mensagem de cobertura MARS no console.
      3. Se cobertura insuficiente → GC.
      4. Se cobertura OK mas alvo no plano galáctico → ainda preferir GC.
      5. Caso contrário → MGC.

    Tipo de alvo                    Separar?    Justificativa
    ──────────────────────────────────────────────────────────────────────────
    Galáxias (M51, M101, M104…)     SIM         Estrelas distraem; stretch agressivo precisa
                                                ser feito sem inflar estrelas
    Galáxias com estrutura fina     SIM         Máximo ganho — núcleo brilhante + braços
    (M81, M63, NGC 891)                         fracos beneficiam de processamento separado
    Nebulosas de emissão            SIM         Stretch da nebulosidade infla estrelas;
    (M8, M17, M16, Carina)                      LHE/contraste sem ringing
    Nebulosas de reflexão           SIM         Mesma lógica
    (M78, Iris, Pleiades neb.)
    Nebulosas planetárias compactas SIM         Tratamento separado do disco e estrelas de
                                                campo
    IFN / cirros galácticos         SIM (forte) Sinal fraquíssimo exige stretch extremo —
                                                separação é praticamente obrigatória
    Aglomerados abertos             NÃO         Estrelas SÃO o objeto. Imagem starless seria
    (M6, M7, M11, M45, NGC 6231)                praticamente vazia.
    Aglomerados globulares          NÃO         Densidade estelar extrema confunde o SXT;
    (M22, ω Cen, 47 Tuc)                        e estrelas são o objeto.
    Cometas                         CASO A CASO Depende se quer destacar a cauda da cabeleira
                                                vs estrelas de fundo.

    CAVEAT IMPORTANTE: mesmo em aglomerados, SXT pode ser usado APENAS para
    GERAR star mask (sem usar a starless). É um uso legítimo e poderoso para
    aumentar saturação só nas estrelas — perfeito para mostrar variedade de
    cores em M6 (mistura de azuis quentes e laranjas frias).


  ── 5.6 Recombinação starless + stars — opções PixelMath (NOVA) ──────────────

    Operação                Expression                          Quando usar
    ──────────────────────────────────────────────────────────────────────────
    Screen blending         1 - (1-$T) * (1-stars)              DEFAULT — preserva valores ≤1
    (recomendado)                                               e evita overflow nos núcleos

    Soma simples            $T + stars                          PERIGOSO — overflow em núcleos
                                                                saturados gera ringing.
                                                                Usar SOMENTE se ambas estão muito
                                                                escuras (sub-stretched).

    Soma com clamp          min(1, $T + stars)                  Funciona mas truncamento gera
                                                                bordas duras em núcleos.

    Screen com fator        1 - (1-$T) * (1 - stars*k)          k = 0.7-0.9 para reduzir
                                                                intensidade de estrelas no final
                                                                (alternativa à redução
                                                                morfológica).

    NOTA: nas três primeiras, $T é o starless (aplicação da expressão) e
    "stars" é o nome da imagem de estrelas. Em PixelMath multi-canal, mesma
    expressão se aplica simetricamente a R, G, B.

  6. MÁSCARAS — Guia de uso (NOVA SEÇÃO)
════════════════════════════════════════════════════════════════════════════════

  Filosofia: máscara branca = processo passa, máscara preta = processo
  bloqueado, cinza = parcial. Inverter (Ctrl+Shift+I com máscara ativa) para
  trocar comportamento.

  ── 6.1 Luminance Mask ───────────────────────────────────────────────────────

    Onde usar:
      • NXT seletivo (proteger objeto, reduzir só no fundo) — invertida
      • Curves no objeto sem afetar fundo
      • Saturation sem amplificar ruído colorido do fundo
      • Sharpen sem afetar background ruidoso

    Como criar (imagem RGB stretched):
      1. ChannelExtraction → CIE L*a*b* → Apenas L
         (ou botão "Extract CIE L* Component" da toolbar)
      2. Aplicar HT/STF p/ esticar a máscara — fundo bem escuro, objeto bem
         claro (mais contraste = mais seletividade)
      3. Opcional: Convolution leve (StdDev 1.5-2) para suavizar transições

    Aplicação:
      • Arrastar máscara para barra de máscara da imagem-alvo
      • Imagem fica com tonalidade vermelha onde máscara é preta (=protegido)
      • Inverter máscara se a proteção estiver no lugar errado

    Caveat: máscara não-linear precisa ser regerada se a imagem-base for
    re-esticada. Anotar etapa de geração.


  ── 6.2 Star Mask ────────────────────────────────────────────────────────────

    Onde usar:
      • Aumentar saturação só nas estrelas (excelente p/ aglomerados)
      • Reduzir tamanho de estrelas com MorphologicalTransform
      • Sharpen seletivo evitando estrelas (mask invertida)
      • Em narrowband: evitar halos coloridos de multiplicadores em R/B

    Como criar — método moderno (SXT):
      1. Aplicar SXT na imagem (ou usar a imagem stars já gerada)
      2. Stretch da imagem stars se ainda linear
      3. Aplicar binarization OU curve agressiva p/ tornar quase binária
      4. Convolution leve p/ suavizar bordas

    Como criar — método tradicional (processo StarMask):
      1. Imagem stretched como base
      2. Processo StarMask, ajustar Noise threshold, Scale, Smoothness
      3. Pode precisar combinar 2-3 passes (estrelas brilhantes vs fracas)
         via PixelMath: max(mask1, mask2, mask3)

    Caveat: stars do SXT capturam estrelas até bem fracas. Star mask muito
    inclusivo bloqueia processamento em demasiados pixels. Pode ser preciso
    ajustar com curves para reter só as estrelas que importam para o passo.


  ── 6.3 Range Mask ───────────────────────────────────────────────────────────

    Onde usar:
      • Proteger highlights (núcleos brilhantes) em curves agressivas
      • Selecionar faixa específica de brilho (mid-tones de uma galáxia)
      • Criar máscara de núcleo para HDR-like processing

    Como criar:
      1. Processo RangeSelection (precisa imagem stretched)
      2. Ativar Real-Time Preview
      3. Ajustar Lower limit e Upper limit visualmente
      4. Smoothness: 0.10-0.30 para transições suaves

    Caveat: "Only viably works on non-linear (stretched) images".
    Não tentar usar antes do MAS.


  ── 6.4 Onde encaixar máscaras no workflow ───────────────────────────────────

    Etapa                                        Máscara recomendada
    ──────────────────────────────────────────────────────────────────────────
    NXT linear (sem star sep)                    Nenhuma — NXT já é seletivo
    NXT em starless                              Nenhuma — sem estrelas, sem máscara
    NXT em stars                                 Nenhuma — todas são estrelas
    Curves criativas no starless                 Luminance Mask
    Curves criativas no canal final              Luminance Mask
    LHE no L_starless                            Luminance Mask
    ColorSaturation final                        Luminance Mask + opcionalmente Star Mask
    Boost de cor de estrelas                     Star Mask
    Redução de tamanho de estrelas               Star Mask (não invertida)
    Sharpen extra em galáxia/nebulosa            Luminance Mask + Star Mask invertida
    Limpeza de ruído colorido residual           Luminance Mask invertida

    Para aglomerados (sem star separation): a Star Mask vira a principal
    ferramenta. Curves + ColorSaturation com Star Mask é onde se realça a
    diversidade de cor estelar.

  Regra                                              Motivo

  SPFC ANTES de MGC                                  MGC precisa de fluxo físico para
                                                     comparar com MARS

                                                     funciona em imagem colorida

  ChannelCombination para R+G+B lineares             LRGBCombination opera em Lab —

                                                     correta em espaço Lab

  NÃO aplicar star separation em aglomerados         Estrelas são o objeto;
                                                     starless ficaria vazia

  8. PREMISSAS E LIMITAÇÕES
════════════════════════════════════════════════════════════════════════════════

  • WBPP configurado com plate solving habilitado e parâmetros corretos de
    escala de placa por rig.

  • Todas as aplicações de BXT/NXT/SXT/MAS são em MASTERS INTEGRADOS, nunca
    em subs individuais. (Aplicar em subs gera variação de output por
    inferência ML e degrada a integração.)

  • XMARS database baixado (~1.35 GB) e configurado no MGC.
    Versão DR1 v1.1 (mar/2025) ou superior.

  • Catálogo Gaia DR3/SP instalado localmente para SPFC e SPCC.

  • StarXTerminator v2+ com AI atual instalado.
    Documentar versão do AI nas premissas da sessão p/ reprocessamento.

  • Em céu com poluição luminosa forte, NXT linear pode causar manchas após
    o stretch — nesse caso mover NXT para após o MAS.
    (Munhoz Bortle 3-4 — geralmente OK, exceto alvos baixos a sul/SW com
    light dome de SP-MG.)

  • MAS é ferramenta nova (dez/2025) — se resultado não agradar, GHS é
    alternativa robusta e estabelecida.

  • Para narrowband (Ha, OIII, SII): workflow muda — cobertura MARS pode ser
    limitada por filtro, e SPFC requer curvas de transmissão configuradas
    por filtro narrowband.

  • MGC requer verificação de cobertura MARS no tool antes de uso para
    qualquer alvo no hemisfério sul. NÃO confiar em regra fixa de declinação.

  • LinearFit 2 (L → RGB) é opcional se MAS foi aplicado com o mesmo
    Target background em todas as variantes.

  • Star separation NÃO se aplica a aglomerados estelares (5.5). Para
    aglomerados, SXT serve apenas como gerador de star mask.

  • Processo de star separation pode falhar em aberrações severas. Epsilon
    160 = óptica excelente, raramente um problema. Se utilizar com rig com
    coma forte sem correção, espere artefatos.

  • Máscaras NÃO destroem dado — são reversíveis. Pode trocar/remover
    máscara sem reprocessar a imagem.

  • Convenção de nomenclatura sugerida (para evitar confusão com tantas
    imagens intermediárias):
        Alvo_Canal_Estado_Etapa
      Exemplos:
        M51_RGB_linear_postBXTcorrect
        M51_RGB_starless_linear
        M51_RGB_stars_linear
        M51_RGB_starless_stretched
        M51_L_starless_stretched_postLHE
        M51_LRGB_final

  9. RESUMO VISUAL DO FLUXO
════════════════════════════════════════════════════════════════════════════════

  RGB:
    WBPP → BXT Correct Only → ImageSolver → Crop → LinearFit R,G,B →
    ChannelCombination → SPFC → MGC/GC → SPCC → [PONTO DE BIFURCAÇÃO]

    Se NÃO aglomerado:
      → SXT linear → [RGB_starless, RGB_stars]
      RGB_starless:  BXT(0.50-0.60) → NXT(0.85) → MAS → SCNR →
                     LumMask → Curves → ColorSat
      RGB_stars:     NXT(0.40-0.50) → MAS(suave)
      → PixelMath screen blend → RGB_final

    Se aglomerado:
      → BXT(0.40-0.50) → NXT(0.80-0.90) → MAS → SCNR →
        StarMask via SXT → Curves+ColorSat com StarMask → RGB_final

  L:
    WBPP → BXT Correct Only → ImageSolver → Crop → SPFC → MGC/GC →
    [PONTO DE BIFURCAÇÃO]

    Se NÃO aglomerado:
      → SXT linear → [L_starless, L_stars]
      L_starless:    BXT(0.80-0.95) → NXT(0.65-0.70) → MAS →
                     LumMask → Curves+LHE
      L_stars:       NXT(0.30-0.40) → MAS(suave)
      → PixelMath screen blend → L_final
      → LinearFit p/ RGB_final (opcional)

    Se aglomerado:
      → BXT(0.50-0.70) → NXT(0.65-0.70) → MAS →
        Curves leves → L_final

  COMBINAÇÃO:
    L_final + RGB_final → LRGBCombination → final RGB
    → StarMask para tweaks finais de cor/tamanho de estrela (todos os alvos)
    → Curves finais com LumMask
    → ACDNR chrominance com LumMask invertida (opcional)
